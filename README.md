# Smart Computer Lab Monitoring System – Manual Testing

## Project Overview

This repository contains the manual testing artifacts for the **Smart Computer Lab Monitoring System** web application. The project focuses on validating the application's functionality, usability, integration, performance, and browser compatibility to ensure it meets the expected requirements.

## Project Objective

To verify that all application modules meet the specified requirements and ensure users can access and use the application effectively.

## Modules Tested

* Login Module
* Dashboard Module
* Activity Records Module
* Change Password Module
* Student Records Module
* Data Upload Module
* Search / Filter Module
* User Management (Add, Edit, Delete User)
* Navigation Controls (Next Button, Previous Button)

## Testing Scope

### In Scope

* Functional Testing
* Usability Testing
* Integration Testing
* Performance Testing
* Compatibility Testing (Chrome, Microsoft Edge, Opera)

### Out of Scope

* API Validation
* Database Validation
* Retesting

## Test Environment

* **Operating System:** Windows 11
* **Browsers:** Google Chrome, Microsoft Edge, Opera
* **Tools:** Microsoft Excel, Microsoft Word

## Repository Contents

* **Test Plan.docx** – Testing strategy, scope, environment, and approach.
* **Test Cases.xlsx** – Detailed manual test cases for all modules.
* **Bug Report.xlsx** – Defect log with severity, priority, status, and expected vs. actual results.
* **Test Summary.docx** – Overall testing results and project summary.
* **Bug Screenshots/** – Screenshots of identified defects.

## Deliverables

* Test Plan

## Outcome

* Executed end-to-end manual testing across all planned modules.
